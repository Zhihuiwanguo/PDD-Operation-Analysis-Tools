拼多多运营分析工具 v1 产品需求文档
1. 项目名称

拼多多运营分析工具 v1

2. 项目目标

将当前人工运营分析流程工具化。
用户上传拼多多订单、推广、流水及映射表后，系统自动完成清洗、匹配、计算、分层、可视化，并输出标准经营分析结果。

3. 适用对象
当前阶段：单人运营者本人使用
后续阶段：可扩展为团队共用工具
4. v1 产品定位

不是实时 API 系统，而是一个表格导入型经营分析工具。

v1 先解决三个核心问题：

上传多张原始表后自动分析
固化统一经营口径，减少人工反复算表
输出可视化经营结果和异常清单
5. v1 范围
5.1 必做范围

支持以下能力：

上传基础数据表
自动识别并清洗订单数据
自动匹配产品、规格、链接映射
自动计算店铺总盘、链接盘、产品盘、规格盘
输出经营分析结果
输出异常映射和异常数据清单
支持日期范围筛选
支持结果导出
5.2 不做范围

v1 暂不做：

拼多多后台 API 实时接入
多账号权限体系
多人协同编辑
自动从后台抓数
手机端适配优先优化
智能投放建议引擎
自动调价/自动投放

7. 经营口径规则

以下规则必须写入系统配置，作为 v1 固定规则。

7.1 订单有效性规则
有效订单

正常成交且售后状态不是退款成功

无效订单

包括：

已取消
未发货退款成功
已发货退款成功
已收货退款成功
其他退款成功
待确认

售后处理中
单独列示，不并入有效，也不直接判无效

7.2 非经营剔除规则
差价补款商品

不纳入经营分析指标
归入非经营剔除项

7.3 推广费规则
链接经营盘推广费

使用字段：
实际成交花费(元)

店铺总盘推广费

使用：
推广账户每日流水表中的现金支出汇总

7.4 ROI 规则
链接实际ROI

商家实收金额 ÷ 实际成交花费(元)

店铺整体实际ROI

商家实收金额 ÷ 店铺总盘推广费（现金支出汇总）

7.5 平台扣点规则
百补链接

平台扣点 = 商家实收 × 2.4%

非百补（日常）链接

平台扣点 = 用户实付 × 0.6%

7.6 毛利规则
订单侧估算毛利

商家实收 - 产品总成本 - 快递总成本 - 平台扣点

扣推广后贡献毛利（链接层）

订单侧估算毛利 - 实际成交花费(元)

扣推广后贡献毛利（店铺层）

订单侧估算毛利 - 店铺总盘推广费

7.7 盈亏平衡ROI

商家实收金额 ÷ 订单侧估算毛利

8.6 异常清单

需要单独输出：

1）未映射规格

订单中存在，但主数据未匹配成功

2）未映射商品ID

推广或订单里存在，但链接表无映射

3）重复映射

同一商品ID+销售规格ID出现多条映射

4）有订单无推广费

可视为自然单或数据缺口

5）有推广费无订单

投放浪费风险

6）差价补款商品

单独列出，不纳入经营分析

7）待确认订单

单独列出，方便后续复核

数据库：

初版可先 SQLite
后续升级 Postgres

数据处理：

pandas
openpyxl
numpy

图表：

plotly
altair 或 echarts

导出：

xlsxwriter
11.2 为什么推荐这个方案

因为你现在：

没代码经验
需要尽快跑出第一版
重点是分析逻辑，不是复杂工程架构

Streamlit 很适合做这种数据工具型产品。

12. v1 非功能要求
12.1 易用性
上传文件后 1 次点击即可分析
报错信息要明确说明缺什么字段
页面尽量简单，少配置
12.2 稳定性
上传错误文件不能导致系统崩溃
对重复映射、空值、异常金额要能提示
12.3 性能
3万行以内订单表，分析时间控制在可接受范围
页面加载不要过慢
12.4 可扩展性
规则应尽量写成配置，而不是全部写死在代码里
为后续多店、多平台预留字段
